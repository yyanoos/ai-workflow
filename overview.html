AI Workflow

AI가 만들고, 테스트가 지킨다

AI에게 코드를 시키고 → 테스트로 검증하고 → 리뷰로 보완한다

이 3단계를 체계적으로 반복하는 것이 전부

기존 프로젝트

테스트 없는 프로젝트에 안전망 깔기

커맨드 1개: /gen-api-tests

초기 셋업 (각 1회, 검수 후 다음으로)

  1. 0 분석 — 서브에이전트가 프로젝트 전체 파악 → PROJECT_CONTEXT.md
  2. 1 스캔 — API 엔드포인트 전수 파악 → endpoints.json

엔드포인트 1개씩 반복

  1. 2 생성 — 통합 테스트 작성 (integration/ 디렉토리)
  2. 3 리뷰 — 서브에이전트가 비즈니스 의도 + 엣지케이스 검증
  3. 4 실행 — 테스트 돌리고 결과 보여줌
  4. ✓ 검수 — 사람이 확인 → 다시 /gen-api-tests
목표: 모든 API × 2+ 테스트 케이스 · 매 엔드포인트마다 사람 검수
신규 기능

테스트 먼저, 구현은 AI에게

  1. 1 스펙 — 뭘 만들지 정의
  2. 2 테스트 작성 — 실패하는 테스트부터 작성
  3. 3 구현 — AI가 테스트 통과하도록 구현
  4. 4 리뷰 + 보충 — 서브에이전트 리뷰 → 테스트 보강
핵심: 매 단계 사람이 /next로 진행 (AI 자동진행 없음)

핵심 원칙

AI가 만든 코드는 테스트를 통과해야만 신뢰한다

생성한 AI와 리뷰하는 AI를 분리한다 (서브에이전트)

모든 진행은 사람이 한 단계씩 제어한다
